CHROME WEB STORE — PRIVACY PRACTICES ANSWERS
=============================================

SINGLE PURPOSE
--------------
Shuffle the user's Suno music library in a randomised order with a persistent queue that survives page reloads.


PERMISSION JUSTIFICATIONS
--------------------------

storage
  Saves the shuffle queue, playback position, played-song history, overlay position, and minimised state to chrome.storage.local so the queue persists across page navigations and browser restarts.

tabs
  Used to locate the active suno.com tab so the popup can send messages to the content script (start shuffle, resume, stop). No tab data is read or stored.

scripting
  The content script (content.js) is injected into suno.com pages to scrape song metadata from the DOM and render the floating media player overlay.

host permissions (suno.com/*, cdn1.suno.ai/*)
  suno.com — the extension operates exclusively on this site; the content script is injected here and the popup communicates with it.
  cdn1.suno.ai — audio files are streamed directly from this CDN using the song UUIDs scraped from the page.

remote code
  This extension does not execute any remote code. All JavaScript runs from scripts bundled inside the extension package (content.js, background.js, popup.js). Audio files are streamed as media data, not executed as code.
